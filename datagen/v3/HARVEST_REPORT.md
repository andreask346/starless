### DELTA vs what v2 actually sampled

v2's `sample_psf_family` was run 20 000 times to get its true sampling distribution (50% jittered real presets, 50% log-uniform 1.4–12 px).

| quantity | v2 sampled (p10/med/p90) | v3 measured (p10/med/p90) | delta |
|---|---|---|---|
| FWHM px | 1.57 / 2.18 / 7.82 | 1.88 / 2.98 / 4.66 | v2 median is 0.73x reality |
| elongation a/b-1 | 0.09 / 0.27 / 0.40 | 0.09 / 0.30 / 0.66 | v2 median is 0.92x reality |

* **75%** of v2's draws land inside the measured 5–95% band (1.53–5.88 px); **8%** are sharper than any real star and **17%** are fatter than the fattest 5%.
* **The root cause is a unit bug in the v2 preset table.** `datagen/calibrate_psf.py` down-samples any frame wider than 4000 px before fitting (`step = ceil(max(h,w)/4000)`, i.e. step 2 on his 7008 px frames) but writes the resulting FWHM into `psf_presets.json` in DOWN-SAMPLED pixels — and `sample_psf_family` then consumes those numbers as FULL-RES pixels. The preset table's 1.52–2.23 px is therefore ~half of the truth; measured full-res median is **2.98 px** ~ 2x the preset median. Every v2 image built from a preset had stars half the right size.

## 2. Measured noise model

* `datagen/v3/noise_model.json` — 231 sources, 102593 star-free 48x48 tiles.
* Model: `sigma(S) = sqrt(a*S + b^2)` per channel (same functional form v2 uses), plus a 9x9 normalised autocorrelation kernel and the R/G/B correlation coefficients.

| parameter | R | G | B |
|---|---|---|---|
| a | 0.000442 [1.3e-05–0.0047] | 0.000242 [5e-07–0.0023] | 0.000381 [4.4e-06–0.0038] |
| b | 0 [0–0.0078] | 0 [0–0.0032] | 0 [0–0.0022] |
| sigma_med | 0.00156 [7e-06–0.015] | 0.00084 [3.3e-06–0.006] | 0.000859 [1.5e-06–0.0058] |
| sigma_rel | 0.0941 [0.0075–0.31] | 0.0983 [0.0087–0.31] | 0.139 [0.012–0.48] |

* **Use `sigma_rel`** (= sigma at the background / background level). The de-stretch gamma differs per frame, so absolute sigma is not comparable across sources; the relative figure is, and it is what a sampler should draw. The `a`/`b` split is ill-conditioned on 68% of frames — over the narrow signal range of a single sky the shot and read terms are degenerate, and the fit puts everything in `a`. Draw `sigma_rel` and use `a`/`b` only for the signal-dependent slope.
* Apples-to-apples amplitude check at the same background anchor (S = 0.02, the de-stretch target): v2 draws sigma/S = 0.043 / 0.129 / 0.302, measured is 0.009 / 0.098 / 0.308 (G). **v2's noise AMPLITUDE is essentially right — its noise TEXTURE is not.**
* **Spatial correlation**: 1/e correlation length = **1.75 px** (from 200 sources); equivalent Gaussian grain sigma ~ 0.874.
* **Channel correlation** (mono frames excluded: 10 sources): rho(R,G) = 0.31, rho(R,B) = 0.21, rho(G,B) = 0.37.

## 4. The v3 thesis (where reality differs from v2)

1. **v2's real-optics presets are half-scale.** Measured full-res FWHM median **2.98 px** vs the preset table's 1.52–2.23 px (fitted on 2x down-sampled frames but used as full-res). Half of v2's training images therefore had stars roughly half the correct width — a systematic mismatch with the census frame the model is scored on.
2. **The other half of v2's draws are a log-uniform 1.4–12 px guess** with a p90 of 7.8 px; reality's p90 is 4.7 px. Capacity is spent on optics he does not own.
3. **Elongation is systematically present and field-dependent.** Measured a/b-1 median 0.30, and ellipticity rises from centre to corner (table above) — v2 samples a single per-image `elong` scalar with no measured field map.
4. **Grain is always correlated.** v2 renders white noise 31% of the time and picks a correlation length at random otherwise; the measured 1/e length is 1.75 px in every frame measured — his stacks plus his NR pass leave a specific, reproducible grain signature, not a random one.
5. **Channel correlation: v2 got this one about right.** Measured rho(G,B) ~ 0.37 [0.15–0.95] vs v2's U(0, 0.9) (median 0.45). Keep it; the gain is in the spatial term, not this one.
6. **Foreground is a real, hard-edged, structured occluder** — rock spires, ridges, buildings — not a random walk with triangles. This is his niche and no competitor trains for it.

## 5. Suggested v3 knobs (straight from the measurements)

* PSF: sample a real stamp from `psf_stamps.npz` (group by field radius) instead of a Moffat. If a parametric fallback is kept, set `fwhm_lo/hi` to about 1.4 / 7.4 px (log-uniform), NOT 1.4/12.
* Re-fit `psf_presets.json` at full resolution, or multiply the existing `fwhm_*` entries by the down-sample step `calibrate_psf.py` used for that frame.
* Elongation: draw per field bin — centre ~0.16, corner ~0.33 (a/b - 1), rather than one per-image scalar.
* Noise: keep the amplitude, force the texture — set `noise_spatial_p = 1.0` and a grain kernel sigma ~0.874 (1/e length 1.75 px). Keep `noise_chan_rho_hi` as is.
* Silhouettes: replace `render_silhouette` with a real mask draw from `datagen/v3/silhouettes/`, and raise `silhouette_p` from 0.30 to ~0.42 (the measured rate at which his frames contain foreground).

## 6. Known limitations of this library

* Stamps are 33x33, so **halos, diffraction spikes and the far wings of bright stars are not captured** — 65 px stamps were tried first and the isolation rule rejected >99% of detections in his dense fields. Bright/saturated behaviour still has to come from the v2 monster-star machinery.
* Saturated stars are excluded by construction, so the library is biased toward mid-brightness stars; that is the right bias for measuring shape, not for measuring dynamic range.
* FWHM/noise are measured after a *heuristic* de-stretch. The gamma is recorded per stamp/source; frames that were already linear (`gamma = 1.0`) are the cleanest subset and can be filtered on.
* Panorama and stitched sources are resampled by the stitcher, which genuinely widens their PSF — that is real for his final images but is not the native optical PSF. `source` is stored per stamp so these can be split out.
* The same physical star can appear in several frames of one shoot; group counts are stamp counts, not unique stars.
* The centre/mid/corner counts are NOT a density measurement: each frame is sampled with 5 analysis windows (centre + 4 corners), so the corner bin is over-represented by construction. Compare bins to each other, not their absolute counts.
* For stitched panoramas there is no single optical axis, so the field radius of a stamp is only meaningful for single-frame sources. v2 already models this with its `virtual_field` geometry; the per-bin numbers here are therefore a lower bound on the true centre-to-corner trend.

## 8. Resuming / extending the harvest

The harvest scripts are copied to **`datagen/v3/tools/`** (with the manifests); the working directory with the per-source shards is the session scratchpad:

```

```

The shipped assets above are self-contained — only the ability to *extend* the harvest incrementally depends on that scratchpad (`shards/`, ~1.7 MB per source). Everything is append-only + resumable:

```
harvest.py       PSF stamps + noise, one shard per source, manifest harvest_done.txt
harvest_sil.py   silhouette masks, manifest sil_done.txt
build.py         re-aggregates everything into datagen/v3 (safe to run mid-harvest)
```

Re-running either harvester skips every source already listed in its manifest, so an interrupted run continues where it stopped; `build.py` can be re-run at any point to refresh the assets and this report.

Exact commands (from the `v3harvest` directory):

```
set PYTHONIOENCODING=utf-8
..\..\..\..\Documents\"For Claude"\starless\.venv\Scripts\python.exe harvest.py            # resumes the PSF/noise pass
python harvest_sil.py --list sil_sources2.txt   # resumes silhouettes
python build.py                                 # re-aggregate + report
```

State on disk: `harvest_done.txt` / `sil_done.txt` (one source path per line, fsynced after each file), `harvest_records.jsonl` / `sil_records.jsonl` (one JSON record per source), and `shards/*.npz` (one stamp shard per source). Deleting a line from a manifest re-does that source; deleting a manifest re-does everything. Widening coverage = raise `per_dir` in `make_lists.py` / `make_sil2.py` and re-run.
